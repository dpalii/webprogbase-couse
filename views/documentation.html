Version: v1

Resources

Users:
  • GET /api/v1/users - returns paginated JSON array of users. URL queries:
    • limit: desired amount of entries
    • offset: amount of entries you want to skip
    • searchword: filter applied to users' logins
  • GET /api/v1/users/{id} - returns JSON object that contains user with id = {id}.
  • POST /api/v1/users - register new user. Returns created user. Reqires such input:
    • username - user login
    • pass - user password
    • pass2 - password confirmation
  • PUT /api/v1/users/{id} - update user with id = {id}. Returns updated user. Pass in 'modification' object with data.
  • DELETE /api/v1/users/{id} - delete user with id = {id}. Returns deleted user.
Products:
  • GET /api/v1/products - returns paginated JSON array of products. URL queries:
    • limit: desired amount of entries
    • offset: amount of entries you want to skip
    • searchword: filter applied to products' names
  • GET /api/v1/products/{id} - returns JSON object that contains product with id = {id}.
  • POST /api/v1/products - create new product. Returns created product. Accepts such input:
    • prodname
    • price
    • prodpic
    • inStock
    • category - id of category the product relates to
  • PUT /api/v1/products/{id} - update product with id = {id}. Returns updated usproducter. Pass in 'modification' object with data.
  • DELETE /api/v1/products/{id} - delete usproducter with id = {id}. Returns deleted product.
Categories:
  • GET /api/v1/categories - returns paginated JSON array of categories. URL queries:
    • limit: desired amount of entries
    • offset: amount of entries you want to skip
    • searchword: filter applied to users' logins
  • GET /api/v1/categories/{id} - returns JSON object that contains category with id = {id}.
  • POST /api/v1/categories - register new category. Returns created category. Reqires such input:
    • name
  • PUT /api/v1/categories/{id} - update category with id = {id}. Returns updated category. Pass in 'modification' object with data.
  • DELETE /api/v1/categories/{id} - delete category with id = {id}. Returns deleted category.

Authorization

This API uses Basic authorization. To access data you should send authorization data with each request. All requests return current user in user field.

Errors

Errors are sent in response body in JSON format, with error message stored in err field. Such error codes are possible:

  • 400 - bad input data
  • 401 - user is unauthorized
  • 403 - user has no access to provided resource
  • 409 - conflicting data
  • 500 - server error
